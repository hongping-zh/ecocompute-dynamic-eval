## 快速入门

### 什么是 EcoCompute？

EcoCompute 是一个综合性的 AI 模型评估和成本优化平台，帮助您：

- **对比 AI 模型**：横向比较性能、成本和碳足迹
- **估算成本**：计算不同 API 提供商和硬件配置的费用
- **优化支出**：为您的使用场景找到最具性价比的方案
- **减少碳排放**：获得环保型模型推荐

### 快速开始（3 步）

1. **访问网站**: https://hongping-zh.github.io/ecocompute-dynamic-eval/
2. **选择视图**: 点击侧边栏在不同功能间切换
3. **开始探索**: 使用计算器或排行榜分析您的选项

无需注册！所有功能免费，直接在浏览器中使用。

## 模型排行榜

### 概述

排行榜展示 22+ 个 AI 模型的实时性能指标，允许您根据优先级对比和排序模型。

### 核心功能

#### 1. **可排序列**

点击任意列标题对模型排序：
- **准确率 (Accuracy)**: 基准测试性能（越高越好）
- **速度 (Speed)**: 执行时间（秒）（越低越好）
- **成本 (Cost)**: 预估月度成本（美元）（越低越好）
- **碳排放 (Carbon)**: CO₂ 排放量（kg/天）（越低越好）
- **能效 (Efficiency)**: 每瓦特处理的 token 数（越高越好）
- **综合评分 (Score)**: 基于您的权重配置的综合得分（越高越好）

#### 2. **数据置信度徽章**

每个模型显示彩色徽章，表示数据质量：
- 🟢 **实测 (Measured)** (绿色): 来自官方基准测试的直接测量
- 🟡 **估算 (Estimated)** (琥珀色): 基于相似硬件的计算值
- 🔵 **研究 (Research)** (蓝色): 学术论文或第三方研究

**悬停在徽章上**查看详细溯源信息：
- 数据来源
- 测量方法
- 最后更新日期
- 引用链接

#### 3. **可配置评分权重**

自定义模型排名方式：

1. 点击 **"Scoring Weights"** 按钮（滑块图标）
2. 调整 5 个滑块以匹配您的优先级：
   - **准确率 (Accuracy)**: 模型性能有多重要？
   - **速度 (Speed)**: 响应时间有多关键？
   - **成本 (Cost)**: 您对预算有多敏感？
   - **碳排放 (Carbon)**: 环境影响有多重要？
   - **能效 (Efficiency)**: 您对能源效率有多看重？
3. 观察 **Score** 列实时更新

**示例配置**：
- **初创公司（成本优先）**: 成本 40%，碳排放 30%，准确率 20%，速度 10%，能效 0%
- **企业（性能优先）**: 准确率 50%，速度 30%，成本 10%，碳排放 5%，能效 5%
- **绿色 AI**: 碳排放 40%，能效 30%，成本 20%，准确率 10%，速度 0%

#### 4. **RTX 5090 筛选**

切换 **"RTX 5090 Only"** 按钮，仅显示针对 NVIDIA 最新 GPU 优化的模型。

#### 5. **AI 分析**

选择多个模型（复选框）并点击 **"Analyze with AI"** 获取：
- 对比优势和劣势
- 使用场景推荐
- 成本效益分析

*注意：需要在设置中配置 Google Gemini API 密钥*

## 成本计算器

### 概述

计算器估算不同硬件和 API 配置的月度成本和碳排放。

### 基础使用

#### 步骤 1：配置硬件

1. **硬件类型**: 选择 GPU 型号（RTX 5090、H100、A100 等）
2. **GPU 数量**: GPU 数量（1-100）
3. **每日小时数**: 每天使用小时数（1-24）
4. **PUE**: 电力使用效率（1.0-2.5）
   - 1.0 = 完美效率（理论值）
   - 1.2 = 优秀（现代数据中心）
   - 1.5 = 平均水平
   - 2.0+ = 较差（老旧设施）

#### 步骤 2：配置 API 使用

1. **API 模型**: 选择提供商（DeepSeek、GPT-4o、Claude、Gemini 等）
2. **每日 Token 量**: 每天的 token 使用量
   - 小型聊天机器人: 10万-50万
   - 中型应用: 100万-500万
   - 大型企业: 1000万+

#### 步骤 3：查看结果

计算器显示：
- **每日成本**: 每天的 API 费用
- **月度成本**: 预计月度支出
- **碳影响**: 每日 CO₂ 排放量（kg）
- **能耗**: 每天消耗的 kWh

### 高级功能

#### 1. **对比模式**

并排对比两个配置：

1. 点击 **"Compare"** 按钮
2. 配置 "Plan B" 设置
3. 查看成本差异和节省百分比

**使用场景**：
- 自建 vs API 对比
- 不同 GPU 型号对比
- 批量折扣场景

#### 2. **自定义公式**

创建高级计费逻辑：

1. 点击 **"Custom Formula"** 开关
2. 使用可用变量输入公式：
   - `tokens`: 每日 token 数
   - `input_price`: 输入价格（每百万 token）
   - `output_price`: 输出价格（每百万 token）
   - `gpu_count`: GPU 数量
   - `hours`: 每日小时数
   - `pue`: PUE 值
   - `power`: GPU 功率（瓦特）

3. 使用高级函数：
   - `IF(条件, 真值, 假值)`: 条件逻辑
   - `MIN(a, b)`: 最小值
   - `MAX(a, b)`: 最大值
   - `ROUND(值, 小数位)`: 四舍五入

**示例公式**：

```javascript
// 免费额度：前 100 万 token 免费
IF(tokens > 1000000, (tokens - 1000000) * input_price / 1000000, 0)

// 批量折扣：超过 500 万 token 打 9 折
tokens * input_price / 1000000 * IF(tokens > 5000000, 0.9, 1.0)

// 成本上限：每月最多 500 美元
MIN(tokens * input_price / 1000000 * 30, 500)

// 最低收费：每月至少 10 美元
MAX(tokens * input_price / 1000000 * 30, 10)
```

**实时预览**: 公式预览显示替换实际值后的表达式，帮助您调试语法错误。

#### 3. **模板**

使用预配置场景快速开始：

1. 点击 **"Templates"** 按钮
2. 浏览分类：
   - 软件工程
   - 客户支持
   - 内容创作
   - 研究
   - 教育
   - 企业
3. 点击模板加载其配置

**热门模板**：

#### 4. **分享与导出**

**分享配置**：

2. 复制生成的 URL

## 历史记录管理

### 概述

计算器自动将您的配置保存在本地（无需账户）。

### 功能

#### 1. **自动保存**

每次配置更改都会自动保存到浏览器的本地存储。如果您关闭标签页后返回，设置会自动恢复。

#### 2. **手动保存到历史**

1. 配置计算器
2. 点击 **"Save"** 按钮（琥珀色图标）
3. 带时间戳的条目添加到历史

#### 3. **查看历史**

1. 点击 **"History"** 按钮（时钟图标）
2. 面板显示最近 50 条记录
3. 每条记录显示：
   - 时间戳
   - 月度成本
   - 每日 CO₂

#### 4. **从历史恢复**

1. 打开历史面板
2. 找到所需条目
3. 点击 **"Restore"** 按钮
4. 配置立即加载

#### 5. **导出历史**

1. 打开历史面板
2. 点击 **"Export"** 按钮
3. 下载包含所有条目的 JSON 文件
4. 用于备份或迁移

#### 6. **导入历史**

1. 打开历史面板
2. 点击 **"Import"** 按钮
3. 选择之前导出的 JSON 文件
4. 历史与现有条目合并

#### 7. **清除历史**

1. 打开历史面板
2. 点击 **"Clear All"** 按钮
3. 确认删除
4. 所有历史删除（无法撤销）

### 隐私说明

所有历史记录**存储在您的浏览器本地**。我们从不将您的数据发送到任何服务器。清除浏览器数据将删除您的历史记录。

## 技巧与最佳实践

### 成本优化

1. **从便宜的模型开始**: 在升级到 GPT-4 之前先测试 GPT-4o-mini 或 DeepSeek
2. **使用批量折扣**: 整合工作负载以达到折扣层级
3. **优化提示词**: 使用简洁的提示减少 token 使用
4. **缓存响应**: 存储常见查询以避免冗余 API 调用
5. **监控使用**: 使用历史记录跟踪支出趋势

### 碳减排

1. **选择低碳模型**: 在排行榜中寻找绿色徽章
2. **优化 PUE**: 使用现代数据中心（PUE < 1.3）
3. **减少空闲时间**: 在非高峰时段缩减规模
4. **批量处理**: 批量处理请求而非实时
5. **使用高效硬件**: RTX 5090 的 tokens/瓦特比旧 GPU 更好

### 公式编写

1. **逐步测试**: 逐步构建复杂公式
2. **使用预览**: 运行前检查变量替换
3. **处理边界情况**: 使用 IF 避免除以零
4. **记录公式**: 在 JSON 导出中添加注释
5. **与团队分享**: 导出有效的公式以供重用

### 模板管理

1. **描述性命名**: 使用清晰的名称，如 "支持机器人-100万-每日"
2. **添加上下文**: 包括团队规模、行业、工作负载模式
3. **定期更新**: 随着定价变化刷新模板
4. **分享成功**: 提交节省成本的模板
5. **分享前测试**: 验证计算准确

## 故障排除

### 计算器问题

**问题**: 结果似乎不正确
- **解决方案**: 检查所有输入是否有效（正数）
- **解决方案**: 验证 API 模型定价是否最新
- **解决方案**: 清除浏览器缓存并重新加载

**问题**: 公式验证失败
- **解决方案**: 检查语法 - 正确使用括号
- **解决方案**: 确保所有变量拼写正确
- **解决方案**: 先用更简单的公式测试

**问题**: 历史记录未保存
- **解决方案**: 检查浏览器是否允许本地存储
- **解决方案**: 确保您不在隐身/隐私模式
- **解决方案**: 如果存储配额超限，清除旧数据

### 排行榜问题

**问题**: 模型排序不正确
- **解决方案**: 再次点击列标题以切换方向
- **解决方案**: 刷新页面重置排序状态

**问题**: AI 分析不工作
- **解决方案**: 在设置中添加 Google Gemini API 密钥
- **解决方案**: 检查 API 密钥是否有足够配额
- **解决方案**: 验证互联网连接

**问题**: 溯源工具提示未显示
- **解决方案**: 在徽章上悬停更长时间
- **解决方案**: 尝试不同的浏览器
- **解决方案**: 禁用阻止工具提示的浏览器扩展

### 导出/导入问题

**问题**: JSON 导出下载空文件
- **解决方案**: 确保您至少有一条历史记录
- **解决方案**: 尝试不同的浏览器
- **解决方案**: 检查浏览器下载设置

**问题**: 导入失败并出错
- **解决方案**: 验证 JSON 文件有效（未损坏）
- **解决方案**: 确保文件是从 EcoCompute 导出的
- **解决方案**: 尝试从源重新导出

**问题**: 报告图片生成失败
- **解决方案**: 检查 html2canvas 库是否加载（查看浏览器控制台）
- **解决方案**: 尝试使用打印/PDF 代替
- **解决方案**: 禁用阻止脚本的浏览器扩展

### 性能问题

**问题**: 页面加载缓慢
- **解决方案**: 清除浏览器缓存
- **解决方案**: 禁用不必要的浏览器扩展
- **解决方案**: 使用现代浏览器（Chrome、Firefox、Edge）

**问题**: 计算器输入时卡顿
- **解决方案**: 关闭其他标签页以释放内存
- **解决方案**: 禁用实时数据模拟
- **解决方案**: 如果有 50+ 条记录，清除历史

## 常见问题

### 一般问题

**问: EcoCompute 是免费的吗？**  
答: 是的！所有当前功能都是免费的。我们计划未来提供 Pro 功能，但核心计算器和排行榜将保持免费。

**问: 我需要创建账户吗？**  
答: 不需要。所有功能都在您的浏览器中运行，无需注册。

**问: 我的数据是私密的吗？**  
答: 是的。所有数据都存储在您的浏览器本地。我们不收集或传输任何个人信息。

**问: 我可以离线使用 EcoCompute 吗？**  
答: 部分可以。应用从 GitHub Pages 加载，但加载后，大多数功能可离线工作。AI 分析需要互联网。

### 定价与成本

**问: 成本估算有多准确？**  
答: 我们使用 API 提供商的官方定价，每月更新。硬件成本是基于云提供商费率的估算。请始终与您的实际提供商验证。

**问: 为什么成本与我的实际账单不同？**  
答: 估算不包括：
- 网络出口费用
- 存储成本
- 支持合同
- 批量折扣（除非在公式中）
- 地区定价差异

**问: 我可以添加自定义定价吗？**  
答: 可以！使用自定义公式实现您的特定定价模型。

### 模型与数据

**问: 模型数据多久更新一次？**  
答: 我们每月更新基准测试。查看溯源工具提示中的"最后更新"日期。

**问: 我可以请求添加模型吗？**  
答: 可以！打开 GitHub Issue，包含：
- 模型名称和版本
- 基准数据来源
- 硬件规格
- 测量方法

**问: 为什么有些指标是"估算"的？**  
答: 当官方基准测试不可用时，我们基于以下估算：
- 相似的硬件配置
- 供应商规格
- 学术研究
- 社区报告

### 功能

**问: 我可以对比超过 2 个配置吗？**  
答: 目前限制为 2 个（Plan A vs Plan B）。使用历史记录保存多个配置并手动对比。

**问: 我可以导出数据到 Excel 吗？**  
答: 导出为 JSON，然后使用转换器或编写脚本转换为 CSV/Excel。

**问: 你们会添加更多 API 提供商吗？**  
答: 会的！随着新提供商获得市场份额，我们会添加。通过 GitHub Issues 请求。

**问: 我可以在我的网站上嵌入计算器吗？**  
答: 目前还不行，但在我们的路线图上。现在请链接到我们的托管版本。

## 术语表

**API**: 应用程序编程接口 - 以编程方式访问 AI 模型的方式

**碳足迹**: 能源消耗产生的总 CO₂ 排放量

**综合评分**: 基于您的优先级的所有指标的加权平均值

**GPU**: 图形处理单元 - 运行 AI 模型的硬件

**kWh**: 千瓦时 - 能源消耗单位

**PUE**: 电力使用效率 - 数据中心效率指标（1.0 = 完美）

**溯源**: 数据的来源和历史 - 指标来自哪里

**模板**: 常见使用场景的预配置计算器设置

**Token**: AI 模型处理的文本单位（约 4 个字符）

**Tokens/瓦特**: 能效指标 - 越高越好
